# 本地运行与打包（Mac / Windows）

> 剪切板管理器 `shijinze-copy`（Tauri v2 + TypeScript）。
> 本文件汇总**本地**开发、编译、打包的所有命令与关键名字。
> GitHub 云端自动打包见 [`GitHub构建流程.md`](./GitHub构建流程.md)。

---

## 0. 关键名字速查

| 名称 | 值 | 说明 |
|---|---|---|
| 产品名 productName | `shijinze-copy` | App 名 / 安装包名 / **进程名（活动监视器/任务管理器）** |
| 应用标识 identifier | `com.shijinze.clipboard-manager` | 唯一 id，辅助功能授权绑定它（**不要随便改**）|
| Cargo 包名 | `clipboard-manager` | dev 模式二进制名（`target/debug/clipboard-manager`）|
| Mac 签名身份 | `Clipboard Manager Dev` | 自签名证书（login 钥匙串），`tauri.conf.json → bundle.macOS.signingIdentity` |
| Mac 安装包 | `shijinze-copy_0.1.0_aarch64.dmg` | Apple Silicon |
| Mac 应用 | `shijinze-copy.app` | |
| Win 安装包(MSI) | `shijinze-copy_0.1.0_x64_en-US.msi` | 64 位 |
| Win 安装程序(EXE) | `shijinze-copy_0.1.0_x64-setup.exe` | 64 位，NSIS |

**产物目录**（编译后）：
- Mac dmg：`src-tauri/target/release/bundle/dmg/`
- Mac app：`src-tauri/target/release/bundle/macos/`
- Win msi：`src-tauri/target/release/bundle/msi/`
- Win exe：`src-tauri/target/release/bundle/nsis/`

## 1. 通用命令（Mac / Windows 一样）

```bash
npm install                     # 安装前端依赖（首次 / package.json 变更后）
npm run tauri dev               # 开发模式运行（热重载；改 Rust 会自动重编）
npm run build                   # 仅前端：类型检查 + 构建（tsc && vite build）
cd src-tauri && cargo check     # 仅后端：Rust 编译检查（不打包，快）
npm run tauri build             # 打正式安装包（Mac 出 .dmg / Win 出 .msi+.exe）
```

> 说明：`npm run tauri build` 会自动先跑 `npm run build`（前端），再 `cargo build --release`（后端），最后打包。

## 2. macOS

### 环境准备（一次性）
```bash
xcode-select --install                                  # Xcode 命令行工具
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh   # Rust
# Node.js：官网下载 LTS，或 brew install node
```

### 每个新终端先加载 Rust（cargo 不在默认 PATH）
```bash
source "$HOME/.cargo/env"
```

### 运行
```bash
source "$HOME/.cargo/env"
npm install          # 首次
npm run tauri dev    # 运行；窗口默认隐藏，按 Cmd+Shift+V 或点菜单栏托盘唤出
```

### 打包（签名 dmg）
```bash
source "$HOME/.cargo/env"
# 若之前的 dmg 还挂载着，先卸载（否则 bundle_dmg.sh 会失败；不要用 /Volumes/clipboard* 通配，无匹配会报错）
hdiutil detach "/Volumes/shijinze-copy" -force 2>/dev/null || true
npm run tauri build
# 产物：src-tauri/target/release/bundle/dmg/shijinze-copy_0.1.0_aarch64.dmg
```

- 打包会用 `Clipboard Manager Dev` 证书**自动签名**（辅助功能授权才绑得住）。证书须在 login 钥匙串里；证书重建方法见 `CLAUDE.md`。
- 未签名/adhoc 会导致辅助功能授权失效、自动粘贴失灵。

### macOS 使用要点
- 默认快捷键 `Cmd+Shift+V`；自动粘贴模拟 `⌘V`
- 无 Dock 图标（菜单栏托盘工具）；毛玻璃无边框窗口
- 自动粘贴需在「系统设置 → 隐私与安全性 → 辅助功能」授权本 App

## 3. Windows

> Mac 上**无法**直接打 Windows 包，需在 Windows 机器上，或用 GitHub Actions（见另一份文档）。

### 环境准备（一次性）
1. **Microsoft C++ Build Tools**：装 [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/)，勾「使用 C++ 的桌面开发」
2. **WebView2 Runtime**：Win10/11 一般自带；没有就去微软官网下
3. **Rust**：下载 `rustup-init.exe` 安装（装完 cargo 自动进 PATH）
4. **Node.js**：官网 LTS 安装包

### 运行 / 打包（PowerShell 或 CMD）
```powershell
npm install
npm run tauri dev       # 运行；按 Ctrl+Shift+V 或右下角托盘唤出
npm run tauri build     # 打包
# 产物：
#   src-tauri\target\release\bundle\msi\shijinze-copy_0.1.0_x64_en-US.msi
#   src-tauri\target\release\bundle\nsis\shijinze-copy_0.1.0_x64-setup.exe
```

### Windows 使用要点
- 默认快捷键 `Ctrl+Shift+V`；自动粘贴模拟 `Ctrl+V`
- 亚克力磨砂窗口；`skipTaskbar`（托盘工具，不在任务栏），从右下角系统托盘找图标
- 未签名 → 首次运行可能弹 SmartScreen「Windows 已保护你的电脑」→「更多信息 → 仍要运行」
- 来源 App 图标为 macOS 专属能力，Windows 上自动降级为类型占位图标

## 4. 常见问题
- **cargo 找不到**：新终端先 `source "$HOME/.cargo/env"`（仅 Mac/Linux；Windows 装完自动进 PATH）。
- **打 dmg 报 `bundle_dmg.sh` 失败**：有同名 dmg 卷挂载着，先 `hdiutil detach "/Volumes/shijinze-copy" -force`。
- 更多踩坑记录见 `CLAUDE.md` 的《问题记录》。
